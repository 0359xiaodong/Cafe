Cafe

A powerful test framework for Android

Download this project as a .zip file Download this project as a tar.gz file

NEWS: Cafe 2.0 is now released!

* Add test project generation by one click.

* Operating across processes is support for all Android system.

Cafe provides the following benefits:

* You can develop test cases across processes by operating them via Cafe.

* You can get & set system state without changed tested application, such as open or close wifi.

* You can get & set fields of tested objects even invoke functions by calling api.

* The framework can handle system running log for some non-ui check point.

* Cafe provides mock function by integrating Android easy mock.

* Cafe provides webapp test api by integrating AndroidDriver.

Quick Start of Cafe

Download Cafe

Then you can run these test cases either on the emulator or on device. You right click the test project and select Run As --> Run As Android JUnit Test. Example of a test case

public class TestCafe extends CafeTestCase {
    private static final String LAUNCHER_ACTIVITY_FULL_CLASSNAME = "com.baidu.chunlei.exercise.MainActivity";
    private static Class     launcherActivityClass;
    private static final String TARGET_PACKAGE                   = "com.baidu.chunlei.exercise";
    static {
        try {
            launcherActivityClass = Class.forName(LAUNCHER_ACTIVITY_FULL_CLASSNAME);
        } catch (ClassNotFoundException e) {
            throw new RuntimeException(e);
        }
    }

    public TestCafe() {
        super(TARGET_PACKAGE, launcherActivityClass);
    }

    @Override
    protected void setUp() throws Exception {
        super.setUp();
    }

    @Override
    protected void tearDown() throws Exception {
        super.tearDown();
    }

    /**
     * NOTICE: 运行case前需要开启cafe_setup.bat
     */
    public void test_sample() {
        remote.setWifiEnabled();
        local.clickOnText("enter");
        assertTrue(local.searchText("welcome"));

        local.clickOnButton("Call");
        // Phone process has been launched, so we should
        // use remote.XXX() to operate oject of other processes.
        remote.clickViewByText("End Call");
        remote.waitForViewByText("Call", 5000);
    }

    @Override
    protected void tearDown() throws Exception {
        super.tearDown();
    }
}
